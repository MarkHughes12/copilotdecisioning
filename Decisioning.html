Copilot Studio: Agent Decision Flowchart

Follow the path to find the right tool for your AI agent development needs.

START HERE

QUESTION 1

Do you need to publish your agent org-wide, to another platform, understand engagement analytics or integrate with other systems?

➡️ YES

RECOMMENDATION

Copilot Studio

For enterprise-grade features like org-wide publishing, multi-channel deployment, analytics, and system integrations, Copilot Studio is the foundational platform.

⬇️ NO

QUESTION 2

Is your primary goal to create an agent that answers questions based only on content from a single SharePoint site or specific libraries/folders?

➡️ YES

RECOMMENDATION

SharePoint Agent

The quickest way to create a focused Q&A experience for a team/project within SharePoint. Ideal for simple, context-specific knowledge management.

⬇️ NO

QUESTION 3

Do you want to build a simple agent for personal/team use (summaries, Q&A from various sources) with a no-code, natural language interface?

➡️ YES

RECOMMENDATION

Agent Builder

A user-friendly, no-code tool in Copilot Studio for business users to create custom agents for personal or team productivity without deep technical expertise.

⬇️ NO

QUESTION 4

Do you need a robust, customized agent with requirements like enterprise data connections, complex workflows, or custom UI?

➡️ YES

RECOMMENDATION

Copilot Studio

A powerful low-code platform for demanding use cases. Use this for integrations, sophisticated conversations, and granular control over agent functionality.

⬇️ NO

QUESTION 5

Does your process require multiple, specialized agents to collaborate and delegate tasks to solve a complex, cross-functional problem?

➡️ YES

RECOMMENDATION

Multi-Agent Orchestration

An advanced feature within Copilot Studio to build a team of collaborating agents for complex, enterprise-level challenges.

➡️ NO

RECOMMENDATION

Copilot Studio

If you've reached this point, your needs can likely be met by the core Copilot Studio platform. It offers a broad range of capabilities for most single-agent scenarios.
